一个前端新人所了解的前端

by 天镶

目前状况：

  • 学校开设前端相关课程较少
  • 很多人对前端了解片面，存在误解
  • 学生在学校、实验室中视野局限

前端人才：

  • 需求量巨大
  • 地位逐渐获得认可
  • 门槛低，精通难
  • 能和产品侃需求
  • 能和设计争界面
  • 能和后台定接口
  • 能和测试修八格

首先应当学什么？

水之积也不厚，则其负大舟也无力——打好基础

  • HTML
  • CSS
  • JavaScript
JavaScript
  • DOM和BOM
  • 面向对象：原型链、prototype、继承
  • 作用域链、闭包、currying、this

JavaScript：

  • 异步控制：Promise、ES6 Generator、ES7 Async
  • 模块化：AMD、CMD、UMD
  • 模式和各种框架：MVC、MVVM、Flux
  • 性能优化：请求合并、sprite等等等等，《Web性能权威指南》
  • 各种ES6新特性：《ECMAScript6 入门》

CSS：

  • 选择器、盒子模型、传统布局：文档流、BFC、IFC、脱离文档流布局
  • 栅格化布局
  • Flexbox、Grid布局
  • 变换和动画：transform、transition、animation

CSS：

  • 自适应和响应式 《响应式Web设计》
  • 浏览器兼容性、常见bug和hack
  • CSS项目：BEM等等
  • 预处理：Less、Sass等等
  • 性能优化：repaint、reflow

HTML：

  • 语义化、可用性、可访问性 《Dont make me think》
  • 浏览器行为
  • 优雅降级、渐进增强、移动先行
  • 搜索引擎优化 SEO

HTML5：

  • Canvas、SVG
  • LocalStorage、SessionStorage、IndexedDB
  • Application Cache
  • Fetch、Service Worker
  • Audio、Video
  • WebSocket、WebRTC
  • 各种移动端API等等等等

我了个血槽这么多...怎么去学呢？

我要开始熬鸡汤了~~

知之者不如好知者，好之者不如乐之者

  • 主动学习
  • 买书是最好的投资
  • 所有知识都是有价值的

好记性不如烂笔头，最好的输入是产出

  • 搭建自己的博客
  • 写文章记录学到的知识
  • 参与各种交流
  • 利用思维导图、脑图帮助学习

纸上得来终觉浅，绝知此事要躬行

  • 搭建一个玩具工厂
  • 利用自己新学的知识造玩具
  • 把漂亮的玩具分享给大家
  • 多去别人的玩具工厂看看，拆拆别人的玩具
  • 将设计图纸转变为玩具

迷时师度，悟了自度

  • 宁为狮尾，不为狐头
  • 向业内人士虚心求教
  • 参加开发者大会
  • 参与开源，阅读优秀代码
  • 善于自己寻找知识，解决问题

不积跬步，无以至千里，不积小流，无以成江海

  • 积累自己的代码，将常用的剥离成小工具
  • 多看、多想、多练
  • 每天读几篇技术文章
  • 每天敲点代码

前端发展迅速，各种框架、思想层出不穷，要保证持续学习，才能跟上脚步

男儿不展风云志，空负天生八尺躯

  • 设计自己的技术理想与路线
  • 勇于尝试不熟悉的技术，逃离技术舒适区
  • 真正的优秀不是比别人优秀，而是比以前的自己优秀
